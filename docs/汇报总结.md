# GitHub CR / CI 工作流汇报总结

> 演示仓库：把前端团队的 Code Review + CI + AI Review 流程跑通，形成可复制的标准流程。

---

## 解决什么问题

- 代码改动没有统一 review 入口，靠口头沟通
- 合并出问题才发现，没有自动化前置拦截
- AI 工具各自为阵，没有接入团队 PR 流程

---

## 当前实现状态

| 目标 | 状态 | 说明 |
|------|------|------|
| push → 自动触发检查 | ✅ | ci.yml，lint + build |
| ESLint 错误 → PR inline comment | ✅ | reviewdog |
| PR 模板 + 审批流程 | ✅ | template + CODEOWNERS |
| AI 自动 Review 并评论 | ✅ | PR-Agent + Gemini |
| AI 交互命令 /ask /improve | ✅ | PR 评论区输入 |
| 切换 AI 后端 | ✅ | 改 pr-agent.yml 两行 |
| CI 失败阻断合并 | ⚠️ | 需配置 Rulesets，免费个人账号私有仓库不强制执行 |
| skip 标签跳过 AI Review | ✅ | workflow if 条件拦截 |

---

## 链路

```
push → PR
  ├─ CI（ci.yml）：lint + build + reviewdog → 通过/失败
  ├─ AI Review（pr-agent.yml）：自动描述 + 中文 review 评论 + 标签
  ├─ Reviewer Approve
  └─ 全部通过 → Merge
```

CI 是硬门禁（需配置 Rulesets），AI Review 只发评论不阻断。

---

## 核心文件

| 文件 | 作用 |
|------|------|
| `.github/workflows/ci.yml` | CI：lint + build + reviewdog |
| `.github/workflows/pr-agent.yml` | AI Review 触发入口，含 skip 标签拦截 |
| `.pr_agent.toml` | AI 审查规则、模型配置、输出语言 |
| `.github/pull_request_template.md` | PR 模板（交付 Checklist） |

---

## AI Review 方案选型

| 方案 | 结论 | 原因 |
|------|------|------|
| **PR-Agent（当前）** | ✅ | 开源免费、多模型、自托管、功能全 |
| CodeRabbit | ❌ | 2025 年 RCE 漏洞 |
| claude-code-action | 备选 | 锁定 Claude，需付费 |

数据安全：自托管模式，代码 diff 只发给配置的 AI API，不经第三方。

---

## 已知限制

| 限制 | 应对 |
|------|------|
| PR-Agent 开源版无法阻断合并 | CI + 人工 Approve 做门禁 |
| `ignore_pr_labels` 对 auto actions 无效 | 在 workflow `if` 条件里拦截 |
| 输出格式（emoji/折叠结构）不可定制 | 只能通过 `extra_instructions` 影响内容 |
| 免费个人账号私有仓库 Rulesets 不强制执行 | 公开仓库或 Team 账号生效 |

---

## 落地建议

1. **试点**：在一个子包加 `ci.yml` + `pr-agent.yml`，跑通链路
2. **扩展 CI**：加 `tsc --noEmit`，类型错误是高频问题
3. **定制审查规则**：编辑 `.pr_agent.toml` 的 `extra_instructions`，补充业务特有规则
4. **切换模型**：改 `pr-agent.yml` 两行环境变量，审查规则不受影响

详细迁移步骤见 [migration-guide.md](migration-guide.md)。
