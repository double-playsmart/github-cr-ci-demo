# 游戏组前端代码审查规范

> 本文件由 Gemini Code Assist 自动读取，作为代码审查的判断依据。
> 修改后 push 即生效，无需其他操作。

## 安全性

- 禁止将用户输入直接赋值给 `innerHTML`、`outerHTML`、`document.write`，防止 XSS 注入
- 敏感信息（API Key、Token、密码）不得出现在代码中
- 所有外部输入必须经过校验后再使用

## 性能

- 避免在循环内进行 DOM 操作或重复调用高开销函数
- 避免无意义的大循环（超过 10000 次且无实际业务意义）
- 事件监听器和定时器必须在组件卸载时清理，防止内存泄漏
- 动画帧回调（requestAnimationFrame）中避免分配大量临时对象

## 代码质量

- 变量、函数、类命名必须有实际含义，禁止单字母命名（循环变量 i/j/k 除外）
- 使用 `const`/`let`，禁止使用 `var`
- 函数职责单一，超过 50 行建议拆分
- 删除无用的 `console.log` 和注释掉的代码块

## 游戏业务专项

- 涉及赔率、倍率、金额的数值计算，检查浮点精度问题（建议整数运算或专用库）
- WebSocket 消息处理必须考虑竞态条件，避免旧消息覆盖新状态
- 游戏状态变更必须通过统一的状态管理入口，禁止直接修改状态对象
- 涉及 `@bc/game-core` 接口调用时，确认方法签名与当前版本一致

## PR 规范

- commit message 必须有实际描述，禁止 `fix`、`update`、`temp` 等无意义提交
- UI 改动必须附截图或录屏

## 审查输出要求

- 评论语言使用中文
- 指出问题时同时给出修改建议
- 区分「必须修改」和「建议优化」并明确标注
